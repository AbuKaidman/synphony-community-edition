Introduction

The chief art of learning, as Locke has observed, is to attempt but little at a time. — Samuel Johnson, 1751.

This web page can help you process language data to produce literacy materials. It supports a wide range of writing systems such as alphabets, abjads, abugidas, and syllabaries, including scripts that are read from right to left. It can produce several output formats, such as:

  • word lists constrained by syllable length
  • creating a separate browser-based literacy tool called SynPhony

The browser-based literacy tool SynPhony extends the capabilities of this page and makes it easy to find words that are restricted to certain letters. This can be useful for producing primers, creating reading assessments, writing stories, or as practise reading material in early literacy efforts.

Browser Requirements

This program (and SynPhony) requires a modern browser that supports HTML 5. Some browsers keep you from loading files from your local computer for security reasons. For local files this page has been tested successfully with Firefox, Safari and Chrome. (Google Chrome must be started with the flag -allow-file-access-from-files in order to work properly offline.) If you have any issues with these three browsers please make sure you're running an up-to-date browser first. This page does not work yet with Internet Explorer and Opera. Hopefully soon.

Language Requirements

The language data must be supplied by you and can come from various sources; either texts, stories, or a word list. It is best if each story is in a separate file. In addition, you will also need to know which digraphs are used in the language, if any. In all cases you should ensure that the language data uses correct spelling, proper punctuation conventions and is encoded in Unicode. If you see this character � anywhere in the output, you need to first convert the data to UTF-8.

Overview

  1. In the Language Data tab there are 4 important steps. For Step 1, provide the language name and ISO-639 code. Next, move through the 3 sub tabs: Texts, Letters, and Words. In the Enter Text tab, import the text files by using the Browse buttons. Move to the Letters sub tab and click the Auto detect button to auto detect the writing system. If it can detect it, the program will also fill in the vowels, consonants and other categories for you automatically. If the language has digraphs you will need to add them manually to the Multigraph textboxes. Visually check to see that all the boxes are filled in correctly. Then, move to the Word Lists sub tab and click the Create Word lists button.
  2. Once the language data has been entered, move on to the Segmenting tab and click the Segment Word List from Language Data tab button to process the word list. If the Residue boxes show up then it is because the language data contains characters which are not listed as a grapheme in the Language Data tab. The Segment Word List from Language Data tab button must be pressed each time you load the page to enable you to use the features in the Output tab.
  3. In the Phonotactics tab you can view the phonotactic analysis as discovered by the program. Phonotactics describe the distribution of letters (or phonemes) within words. Although this tab is only for your information and you don't have to do anything, reviewing the information here can help you spot spelling mistakes or foreign words in your word list due to letters occurring in unexpected positions.
  4. In the Output tab you can select the format for the data output. Press one of the Output buttons to process that group of words into the format you selected. You may copy the resulting data into your computer’s clipboard for using it elsewhere. More information on the various output formats is available in the Output tab beside each option.

About this web page

I hope this software will prove useful in promoting literacy skills in many languages, especially minority languages, around the world. Built-in support is provided for a variety of writing systems, but these have not all been tested yet. I would appreciate feedback from anyone who uses this page with a non-Roman script.

If you use this page and the companion SynPhony program for developing literacy materials I would appreciate hearing about it. Countless hours have gone into developing these tools and the best reward I can think of is hearing how people are actually using them. So, please and let me know how and where you are using it.

If you have the ability and interest to help develop this tool, please and let me know how you would like to contribute.

Browser Issues

This page has been tested and works with Firefox, Safari and Chrome. (Google Chrome can be started with the flag "-allow-file-access-from-files" in order to work properly offline.) I recommended using Firefox. I have not figured out how to get other browsers to read data files when the program runs from the local disk. Hopefully this will change in future versions.

For the SynPhony program, the following browsers have some issues. If the writing system uses zero-spaced characters (most Indic and East Asian scripts, among others) Chrome, Opera, and Safari don't work correctly yet; Firefox and IE9+ work well. For Roman scripts all the major browsers are fine.

Internet Explorer and Google Chrome don't run this file or SynPhony locally (off your hard drive), but have no problem running it from a web server. Firefox and Safari have no problems with local files. So far, only Firefox has support for complex scripts that require Graphite font technology. You can get instructions for enabling Graphite font processing in Firefox here.
